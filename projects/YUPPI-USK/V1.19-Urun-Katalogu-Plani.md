# YUPPI ERP - V1.19 Ürün Kataloğu (Product Catalog) Geliştirme Manifestosu

Sevgili Yapay Zeka Asistanı,
Bu belge, YUPPI ERP projesinin halihazırda stabil olan V1.18 sürümüne entegre edilmek üzere **V1.19 Test Ortamında** yeni bir "Ürün Kataloğu" modülü geliştirmek için hazırlanmış teknik detaylar dokümanıdır.

**NOT:** Lütfen doğrudan canlı sunucu süreçlerini (PM2 "YUPPI" process) bozmadan, test amacıyla kendi yerel sunucunuzda (`npm run dev`) çalışarak aşağıdaki adımları inşa edin. Modül kusursuz çalıştığında asıl sisteme (OrderEntryForm'a) entegre edilecektir.

---

## 1. Modülün Amacı
Kullanıcının şu anda Sipariş giriş ekranında (OrderEntryForm ve OrderItem modeli) manuel ve serbest metin olarak girdiği ürün özelliklerini standart, seçilebilir ve yönetilebilir bir veritabanı haline getirmek. Ürünlerin özelliklerini, belirli fiyat bantlarını ve asgari sipariş miktarını (MOQ) önceden saptayarak sipariş sırasındaki hata payını sıfıra indirmek.

## 2. Prisma Veritabanı Modeli
İlk adım olarak `schema.prisma` dosyasına yepyeni bir `Product` modelini aşağıdaki yapıya (veya uygun benzerine) göre eklemelisiniz:

```prisma
model Product {
  id             Int      @id @default(autoincrement())
  code           String   @unique // Ürün Kodu
  name           String   // Ürün Adı / Kalitesi
  composition    String?  // Kompozisyon (Örn: %100 Pamuk)
  weight         String?  // Ağırlık (Örn: 150 gr/m2)
  width          String?  // En (Örn: 150 cm)
  gtipNo         String?  // Gümrük Tarife İstatistik Pozisyonu (GTİP)
  
  minOrderQty    Float    @default(0) // Asgari Sipariş Miktarı (MOQ)
  unitPriceMin   Float?   // Minimum satış fiyat limiti
  unitPriceMax   Float?   // Maksimum satış fiyat limiti
  
  isActive       Boolean  @default(true)
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt
}
```
*Not: Bu modeli ekledikten sonra `npx prisma db push` işlemini yapıp, test veritabanını güncelleyin.*

## 3. Geliştirilecek Arayüzler (UI)

1. **API Rotaları:**
   - `src/app/api/products/route.ts` (GET, POST)
   - `src/app/api/products/[id]/route.ts` (GET, PUT, DELETE)

2. **Yönetim Ekranları:**
   - Kullanıcıların ürün ekleyip güncelleyebileceği bir sayfa: **`src/app/products/page.tsx`**
   - Bu sayfada mevcut tüm ürünler bir tablo (DataGrid / TableClient) olarak listelenmelidir. Tıpkı "Firmalar (Companies)" tablosundaki arama, düzenleme fonksiyonlarına benzer temiz ve modern bir yapıda tasarlanmalıdır.
   - Sol Sidebar'a (Menüye) "Ürünler" adında yeni bir navigasyon linki (`/products`) eklenmelidir.

3. **Kontrol Mantığı:**
   - Ürün eklenirken MOQ (Asgari Sipariş) ve Fiyat limitleri opsiyonel girilebilir olmalıdır.
   - Yalnızca "Aktif (isActive: true)" olan ürünler daha sonra sipariş formunda çıkacaktır.

## 4. Siparişe Entegrasyon Aşaması
Ürünler modülü kendi içinde (ekleme, silme, listeleme) bağımsız ve tamamen bitmiş aşamaya geldiğinde, `OrderEntryForm.tsx` içerisindeki OrderItem manuel satır girişini "Ürün Seçme (Dropdown veya Autocomplete)" mantığına dönüştürmeniz istenecek. 
Bu aşamada:
- Sipariş formunda ürün seçildiğinde; En, Kompozisyon, Gtip vb. alanlar otomatik dolmalı.
- Girilen Adet, ürünün MOQ (minOrderQty) değerinden düşükse sistem kullanıcıya "Minimum sipariş adedinin altında giriş yaptınız" fonsiyonel uyarısını göstermeli.
